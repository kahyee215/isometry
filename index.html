Key Takeaways

非洲市场的护城河在于与本地移动支付网络的深度整合，Flutterwave 是非洲市场绝对的头部玩家

在非洲，仅靠传统的银行对公接口完全不够，移动钱包的高度碎片化是核心壁垒。

  • 三家公司在肯尼亚、尼日利亚、卢旺达、坦桑尼亚、乌干达、南非、喀麦隆、科特迪瓦、塞内加尔等市场高度重叠，并与 MTN、Airtel、Orange、M-PESA、Moov、Wave 等移动钱包深度绑定。
  • Flutterwave 的本地移动钱包接入广度最强，尤其是在西非法语区；尼日利亚收单渠道覆盖 card、USSD、NQR、Apple Pay、eNaira 等。
  • NALA 的优势来自 2C remittance payout 网络，并通过 Rafiki 向 B2B 支付延伸；Yellow Card 的优势来自 stablecoin-native on/off-ramp 和本地 fiat liquidity。
  • Last-mile 本地渠道是重资产运营，需要处理本地实体合规和流动性资金配置，新玩家切入成本很高。

EUR 覆盖数量很高，但更多是标准化 rails，不作为最强竞争壁垒

在欧洲，三者虽多有覆盖，但更多是 card / bank / SEPA / IBAN 等标准化 rails，差异化弱于非洲本地 mobile money 网络。

  • NALA 和 Flutterwave 对欧洲覆盖很广，但主要依赖可规模化复用的 card、Apple Pay、bank、account、SEPA rails。
  • 欧洲更像 funding / collection side，而不是 last-mile payout 战场。
  • 非洲代表本地渠道深度，欧洲代表标准化资金入口的覆盖广度。

拉美市场的打法：直连国家级即时银行 API

在拉美，Yellow Card 主要靠接入实时支付 API，与其他玩家形成高度同质化的正面竞争。

  • 拉美由政府或央行牵头的即时银行支付网络主导，如巴西 PIX、墨西哥 SPEI、阿根廷 CVU/CBU。
  • B2B 客户会直接对比 API 性能、费率和在当地银行/钱包网络的深度。
  • Yellow Card 和 OpenFX 的 Coming Soon 市场高度重叠，后续执行节奏值得关注。

亚太和中东正成为新的核心增量市场

非洲市场的本地渠道已被 NALA 和 Flutterwave 高度瓜分，APAC 和 MENA 正成为新的增量方向。

  • NALA 已深度绑定南亚侨汇市场，在印度、巴基斯坦、孟加拉对 UPI、bKash、Nagad、Easypaisa、JazzCash 等渠道具备独占性。
  • 中东需要牌照或重量级持牌合作，纯 API 接入通常不够。
  • 阿联酋、沙特、巴林是 Yellow Card 和 Flutterwave 正在建立合规存在的核心区域。

License Coverage

监管门槛：Out 通常比 In 更难

In 侧：合规压力主要在资金汇出国，PSP 可以持有资金传输类牌照，或通过 EMI 等持牌合作伙伴间接完成。

Out 侧：合规压力落在资金接收国，PSP 通常必须自持 PSP / IMTO / 本地银行牌照，或深度绑定本地银行、移动钱包运营商和清算系统。

在外汇管制严格的市场，资金必须在指定银行完成外币结汇，并接受央行额度与真实性审核。

三家 PSP 各自概况

NALA：从面向海外侨民的 remittance app 起家，通过 IMTO / PSP / PSO / MSB / MTL 和 Modulr 等合作向多牌照 PSP 演进。

Flutterwave：全球化全栈 PSP，覆盖 card acquiring、mobile money、bank rails 和 blockchain settlement，监管覆盖范围最广。

Yellow Card：stablecoin-native infra provider，连接本地 fiat rails 和美元稳定币流动性，覆盖 20 个非洲国家、14 种本地货币，处理逾 $6B 交易量。

License 解释表

牌照层级：银行 > 支付系统 > 电子货币 > 支付服务 > 汇款 > 加密资产 > AML 注册；层级越高，业务自主权越大。

牌照名称 适用范围 层级 由谁监管 作用

License / Local Partnership Coverage

自有公司自己持有能直接经营业务的牌照，比如 MTL, PSP, IMTO, EMI, VASP, PI。
注册公司完成监管登记或 AML reporting registration，比如 FinCEN MSB, FINTRAC MSB, FIC / FIU registration；通常不等于完整经营牌照。
